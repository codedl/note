实现过程：  
1. 创建Enhancer增强器对象
2. 通过setSuperclass设置要增强的类，通过setInterfaces设置要增强的接口
3. 通过setCallback设置目标方法拦截器，即可以在目标方法前后进行增强。如果是对接口进行增强，则methodProxy.invoke(实现类对象, objects);如果是对类进行增强，则methodProxy.invokeSuper(obj, objects)，obj为intercept方法的参数。MethodInterceptor为Callback的子接口，可以向上转型成Callback作为setCallback的参数。在代理类的增强方法中将回调转型成MethodInterceptor接口，然后调用接口中声明的intercept方法。所以可以设置成MethodInterceptor。
4. [调用enhancer.create()生成代理类对象](#代理类class)  

原理：通过asm框架技术生成代理类class，然后反射创建代理类对象。代理类中通过获取setCallback时的MethodInterceptor接口为目标方法进行增强。
